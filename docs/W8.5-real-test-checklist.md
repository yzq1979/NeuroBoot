# NeuroBoot v3.0 — W8.5 U 盘真测 checklist

> **用法**：build 好 NeuroBoot.iso → 拷到 Ventoy U 盘 → 用户主系统 F12 选 U 盘 → 进 PE
> → 按本清单逐项测 → 发现 bug 写进 [TODO-v3.0.1-fixes.md](TODO-v3.0.1-fixes.md)
>
> **预计耗时**：每个 section 5-15 分钟；全部 ~60-90 分钟。
>
> **测试硬件要求**：
> - 一台 Win11 主机，已关 Secure Boot，**已禁用 SAC**（Smart App Control，否则 NeuroBoot.exe
>   被 Windows 直接干掉，参考 [KNOWN-ISSUES.md](KNOWN-ISSUES.md)）
> - **有线 USB 鼠标 + 有线 USB 键盘**（PE 无蓝牙 stack）
> - **U 盘 ≥ 8 GB**（NeuroBoot.iso ~3.5-3.9 GB + Ventoy ~10 MB）
>
> **关联**：
> - [TODO-v3.0.md](TODO-v3.0.md) —— v3.0 开发进度
> - [TODO-v3.0.1-fixes.md](TODO-v3.0.1-fixes.md) —— 本测发现的 bug 收集模板

## 6. W6-7：Hooks + Persistent Memory

**前置**：在 U 盘根放 `NeuroBoot.hooks.json` + `NeuroBoot/memories/MEMORY.md`（模板见 `docs/usb-templates/`）。

### 6.1 Hooks

| # | 测试 | 期望 |
|---|---|---|
| 6.1.1 | U 盘根 `NeuroBoot.hooks.json` 加 SessionStart hook 写「machine-info」到 stdout | NeuroBoot 启动后 system prompt 顶部含「# SessionStart hook 注入」+ hook 输出 |
| 6.1.2 | 加 PreToolUse hook（matcher: delete_path）`Write-Error 'no'; exit 2` | AI 试调 delete_path 时拿到「（hook 拒绝）...」tool result，**不会**真执行 |
| 6.1.3 | 加 PostToolUse hook 写 log 到 X:\NeuroBoot\logs\tool-trace.log | 任何工具调用后 log 文件多一行 |
| 6.1.4 | 加 Stop hook | 每轮对话结束（Done 前）跑一次 |

### 6.2 Memory（跨重启）

| # | 测试 | 期望 |
|---|---|---|
| 6.2.1 | U 盘 `NeuroBoot/memories/MEMORY.md` 写「我的机器是 X1 Carbon Gen 11，32GB RAM」| NeuroBoot 启动后 welcome 消息含「MEMORY.md 已加载（XX 字节）」|
| 6.2.2 | 问「我的机器是什么型号」 | AI **不调 read_system_info**，直接答「X1 Carbon Gen 11」（已在 system prompt 顶部）|
| 6.2.3 | 说「记住：我最常出问题的是网络」 | AI 调 `memory(command='create' 或 'str_replace', path='MEMORY.md', ...)` 更新文件 |
| 6.2.4 | 关 NeuroBoot → 重启 PE → 再启 NeuroBoot | 6.2.3 的记录还在（持久化验证）|
| 6.2.5 | 用户试图让 AI 写 `..\\..\\Windows\\System32\\drivers\\etc\\hosts` | AI 收到 `PathTraversal` 错误 → 告诉用户「路径不安全」 |
