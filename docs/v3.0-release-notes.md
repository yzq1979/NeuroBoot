## 新功能（用户视角）

### 1. AI 把工具用得更准

v3.0 重写了**全部 22 个工具的描述**（按 Anthropic 2026 best practices）：4 个必需 section
（When to use / Parameters / Returns / Notes）+ 长度区间 [200, 1500] 字符 + verb_object
命名。配套 `cargo test` 时跑 `assert_v30_description_convention()` 自动检查。

**用户感知**：问「我电脑有几块硬盘」→ AI 不再绕路调 `read_system_info`，直接 `list_disks`。

### 2. 5 个新只读诊断工具

- `list_winre_status` —— 查 Windows Recovery Environment 启用状态 + BCD 默认引导项
- `bitlocker_status` —— 列所有盘的 BitLocker 加密状态 + 保护方式
- `find_large_files` —— 扫指定路径找大文件（按大小降序）
- `read_recent_installs` —— 最近装的 Windows Update KB（QuickFixEngineering 时间线）
- `lookup_error_code` —— 查 BugCheck / Win32 / HRESULT / NTSTATUS 错误码含义

### 3. 5 个新核心诊断 skill

启动加载（progressive disclosure tier 1），AI 看到 summary 后按需调 `load_skill(name)` 拿
完整剧本（tier 2）：

- `/recover-bitlocker` —— KB 触发恢复键流程 + 5 步阶梯
- `/fix-boot-failure` —— 6-tier escalation（WinRE → BCD → SFC/DISM → bootrec → image
  repair → reinstall）
- `/reset-password` —— 账户类型 triage（local / Microsoft）+ EFS/DPAPI 警告
- `/diagnose-slow` —— 6 嫌疑层 + 反模式列表
- `/recover-data` —— 「立刻停写源盘」+ 4 case 分流 + target≠source 防护

### 4. Plan Mode（Cline 风格）

复杂任务（>2 工具 / 含 dangerous / 用户显式要求）AI 先调 `propose_plan` 弹审批模态：
- 含 dangerous 步骤 → **红边框** + 每步「⚠ DANGEROUS」标
- 纯 safe 步骤 → 蓝边框
- Approve → AI 按 steps 依次执行；Reject → AI 重新规划

### 5. 持久化 Memory + Hooks

**Memory**：U 盘 `<root>\NeuroBoot\memories\MEMORY.md` 启动时自动注入 system prompt
顶部。AI 看到「我机器是 X1 Carbon Gen 11」就不会重复调 `read_system_info`。用户说「记住
X」→ AI 调 `memory(command='create' 或 'str_replace', ...)` 自动更新文件。**跨 PE 重启
保留**。Path traversal 8 道防线（绝对路径 / `..` / UNC / NUL byte 全拒）。

**Hooks**：U 盘 `NeuroBoot.hooks.json` 配 4 触发点：
- `SessionStart` —— stdout 注入 system prompt
- `PreToolUse` —— 非零 exit 拒绝工具调用
- `PostToolUse` —— 仅记日志
- `Stop` —— 每轮对话结束

### 6. 错误码 RAG 字典

`lookup_error_code` 升级为分层查询：
- **Phase 1（默认）**: SQLite FTS5 trigram 模糊检索 49 条手工 fixture（覆盖 v1.0.1 hardcoded
  24 条 + 25 新增）；CJK 原生支持（trigram = 3 字符滚动窗口）
- **Phase 2（可选）**: + sqlite-vec 向量检索 + Qwen3-Embedding-0.6B + Microsoft Learn
  全量爬虫（~17k 条 BugCheck / Win32 / NTSTATUS）；RRF k=60 融合排序

返回结果含 `source: "rag" | "hardcoded"` 字段方便 debug。

### 7. 自动化回归测试（30 golden prompts）

`cargo test --features eval run_golden_prompts -- --nocapture` 跑 30 个真实用户场景：
- 8 skill 触发（每个 skill 一次）
- 8 free-form 单工具问题
- 5 多步（3+ 工具）
- 5 dangerous 守护（不应真调危险工具）
- 4 edge case（空 / 长 / 身份 / 歧义）

每个 case 期望：必须 / 禁止的工具调用、必须 / 禁止的 regex 匹配、时间 / token 上限、白
名单工具列表（hallucination 防护）。CI 友好：endpoint 不可达 → 全 Skip，不 fail。

### 8. UX 改进（v2 / Sprint 1 续）

- Markdown 流式渲染（assistant 消息支持加粗 / 列表 / 代码块 / 表格）
- 流式 token 输出（不再一次性给全部答案）
- 「停止生成」按钮（busy 状态）
- 顶栏快捷：⚙ 设置 / 文件 / cmd / 日志 / 重启 / 关机 / 退出（v1.0.1 起电源按钮）
- 6 个快捷问题按钮 + U 盘 prompts.txt 下拉（PE 无 IME 中文输入兜底）

## 已知限制

1. **W5-6 Phase 3 操作步骤**：嵌入模型 + 全量 17k 爬虫数据**非默认打包**进 ISO。用户需
   按 [TODO-v3.0.md](TODO-v3.0.md) 的「W5-6 Phase 3 (operational)」节自己跑：
   - 下载 Qwen3-Embedding-0.6B Q8_0 GGUF（~400 MB，hf-mirror 优先）
   - 跑 `python build-error-db.py --crawl --with-fixtures --embedding-url http://...`
   - 重 build ISO

   未跑 Phase 3 时 RAG 退化为 FTS5-only（49 条 fixture），仍优于 hardcoded 24 条。

2. **WinPE 无 IME**：中文输入只能靠快捷按钮 / U 盘 prompts.txt 下拉框 / MEMORY.md 持久化。
   这是 ADK 硬限制，不会改。

3. **PE 无蓝牙 stack**：蓝牙鼠标 / 键盘进 PE 不识别。必须有线 USB。

4. **eval 框架是 report-only**：默认不让行为差异 fail 测试。Qwen3-4B 偶发选错工具时不
   阻塞 CI。`NEUROBOOT_EVAL_STRICT=1` 强制严格。

5. **Smart App Control**：Win11 25H2 真机 SAC 开启会直接干掉 NeuroBoot.exe。必须禁用
   （见 KNOWN-ISSUES.md）。
